# BÀI 4: Tích hợp API Vận chuyển Bất đồng bộ (Technical Learning Prompts)

## I. Giới thiệu WebClient & Bất đồng bộ

**Vấn đề:**

- Gọi API vận chuyển (Giao Hàng Nhanh, Viettel Post) không thể block thread chính
- Nếu dùng RestTemplate (blocking) → API chậm đợi lâu → User phải chờ lâu
- Giải pháp: WebClient (Spring WebFlux) - Non-blocking, Reactive

**Tại sao cần Learning Prompt:**

- Bạn chưa từng dùng WebClient
- Cần hiểu cơ chế Event Loop, Reactive Streams
- Cần so sánh RestTemplate vs WebClient
- Cần mã nguồn production-ready

## III. Response từ AI - Chi tiết Đầy đủ

### **CẤP ĐỘ 1: Ẩn dụ Nhà Hàng**

```
Tưởng tượng nhà hàng có 2 cách phục vụ:

CÁCH CŨ (RestTemplate - Blocking):
- 1 người bồi bàn xử lý 1 bàn từ đầu đến cuối
- Lấy order → chờ bếp nấu → mang ra → chờ khách ăn → lau bàn
- 1 bàn mất 2 tiếng → phục vụ được 4 bàn/ngày
- Nếu 10 khách đến cùng lúc → cần 10 người bồi bàn

CÁCH MỚI (WebClient - Non-blocking):
- 1 người bồi bàn "thông minh":
  * Lấy order bàn 1 → gọi bếp "Làm order bàn 1"
  * Chuyển sang bàn 2, lấy order → gọi bếp "Làm order bàn 2"
  * ...bàn 10, lấy order
  * Khi bếp nói "Bàn 1 xong" → Mang ra bàn 1
  * Tiếp tục bàn 2, 3, ...
- 1 người bồi bàn phục vụ 10 bàn cùng lúc → 10 khách/ngày/người
- Vừa lấy order vừa chờ, không bao giờ "idle"

EVENT LOOP (Netty):
- Là bộ não quản lý việc nào cần làm kế tiếp
- Nếu bàn 5 chưa sẵn sàng → skip, qua bàn 6
- "Non-blocking" = Không bao giờ chờ, luôn có việc làm

THREAD POOL (RestTemplate):
- = Số lượng người bồi bàn có sẵn
- Mặc định 20-100 người
- Nếu có 10,000 khách → 10,000 người idle chờ → lãng phí lương, lãng phí không gian
```

### **CẤP ĐỘ 2: Reactive Streams & Event Loop**

```
REACTIVE STREAMS (Tập hợp quy tắc):
1. Publisher (nguồn dữ liệu):
   - Ví dụ: Giao Hàng Nhanh API trả response
   - Phát ra 0-1 item (Mono) hoặc 0-n items (Flux)

2. Subscriber (người tiêu thụ):
   - Ví dụ: Code của bạn xử lý response
   - Nhận onNext (dữ liệu), onError (lỗi), onComplete (xong)

3. Operator (biến đổi):
   - map(), filter(), retry(), timeout(), etc.

4. Backpressure:
   - Subscriber nói với Publisher: "Tôi xử lý chậm, gửi cho tôi chậm 1 chút"
   - Tránh Publisher "dội" quá nhiều dữ liệu

EVENT LOOP (Netty):
```

while (true) {
for task in readyTasks {
execute(task)
}
if (noMoreReadyTasks) {
waitForIOEvent() // Non-blocking wait
}
}

```
- Không có "Thread per request"
- 1 thread event loop quản lý 1000+ request
- IO operations không block thread (ghi/đọc network async)

MONO vs FLUX:
- Mono<T>: 0 hoặc 1 giá trị
  * Ví dụ: WebClient.get().retrieve().bodyToMono(DeliveryResponse.class)
- Flux<T>: 0 hoặc nhiều giá trị
  * Ví dụ: Stream dữ liệu liên tục từ server
```

### **Code Java: DeliveryIntegrationService**

```java
// ===================== DTO =====================
@Data
@Builder
public class DeliveryResponse {
    private String trackingId;
    private String status; // PENDING, IN_TRANSIT, DELIVERED
    private String message;
}

@Data
@Builder
public class OrderDeliveryEvent {
    private String orderId;
    private String customerId;
    private String recipientName;
    private String address;
    private String phone;
    private BigDecimal codAmount;
}

// ===================== SERVICE (WEBFLUX) =====================
@Service
@Slf4j
@RequiredArgsConstructor
public class DeliveryIntegrationService {

    private final WebClient webClient;
    private static final String GHN_API_URL = "https://api.giao-hang-nhanh.vn/order/create";
    private static final int TIMEOUT_SECONDS = 5;
    private static final int MAX_RETRIES = 3;
    private static final long INITIAL_BACKOFF_MS = 1000;

    // Cấu hình WebClient
    @Bean
    public WebClient webClient() {
        HttpClient httpClient = HttpClient.create()
            .responseTimeout(Duration.ofSeconds(TIMEOUT_SECONDS))
            .option(ChannelOption.CONNECT_TIMEOUT_MILLIS, 5000)
            .option(ChannelOption.SO_KEEPALIVE, true);

        ExchangeStrategies strategies = ExchangeStrategies.builder()
            .codecs(configurer -> configurer.defaultCodecs()
                .maxInMemorySize(1024 * 1024)) // 1MB max
            .build();

        return WebClient.builder()
            .clientConnector(new ReactorClientHttpConnector(httpClient))
            .exchangeStrategies(strategies)
            .build();
    }

    /**
     * Gửi thông tin đơn hàng sang Giao Hàng Nhanh
     * Retry 3 lần với exponential backoff (1s, 2s, 4s)
     *
     * @param event OrderDeliveryEvent
     * @return Mono<DeliveryResponse>
     */
    public Mono<DeliveryResponse> sendOrderToDeliveryService(
            OrderDeliveryEvent event) {

        log.info("📤 Gửi đơn hàng {} sang Giao Hàng Nhanh...", event.getOrderId());

        return webClient.post()
            .uri(GHN_API_URL)
            .header("Content-Type", "application/json")
            .header("Authorization", "Bearer YOUR_API_KEY")
            .bodyValue(event)

            // Cấu hình timeout 5s
            .retrieve()
            .bodyToMono(DeliveryResponse.class)

            // Retry 3 lần với exponential backoff
            .retryWhen(Retry.backoff(MAX_RETRIES, Duration.ofMillis(INITIAL_BACKOFF_MS))
                .maxBackoff(Duration.ofSeconds(8))
                .jitter(Jitter.random())
                .doBeforeRetry(signal -> {
                    long attempt = signal.totalRetries();
                    long delayMs = (long) (INITIAL_BACKOFF_MS * Math.pow(2, attempt - 1));
                    log.warn("⏳ Retry {} lần cho đơn hàng {} - Backoff: {}ms",
                        attempt, event.getOrderId(), delayMs);
                }))

            // Xử lý lỗi
            .onErrorMap(this::mapDeliveryError)

            // Logging thành công
            .doOnNext(response ->
                log.info("✓ Đơn hàng {} gửi thành công. Tracking ID: {}",
                    event.getOrderId(), response.getTrackingId()))

            // Logging lỗi cuối cùng
            .doOnError(error ->
                log.error("✗ Gửi đơn hàng {} thất bại sau {} lần retry: {}",
                    event.getOrderId(), MAX_RETRIES, error.getMessage()));
    }

    /**
     * Gọi múi đơn hàng (batch multiple orders - Flux example)
     */
    public Flux<DeliveryResponse> sendOrdersInBatch(List<OrderDeliveryEvent> events) {
        log.info("📦 Gửi {} đơn hàng theo batch", events.size());

        return Flux.fromIterable(events)
            .flatMap(event ->
                sendOrderToDeliveryService(event)
                    .doOnError(e ->
                        log.warn("Lỗi đơn hàng {}: {}", event.getOrderId(), e.getMessage()))
                    .onErrorResume(e -> Mono.empty()) // Bỏ qua lỗi, tiếp tục batch
            )
            .parallel()
            .runOn(Schedulers.parallel())
            .sequential();
    }

    // Map lỗi HTTP sang exception tùy chỉnh
    private Throwable mapDeliveryError(Throwable error) {
        if (error instanceof TimeoutException) {
            return new RuntimeException("Timeout kết nối Giao Hàng Nhanh (>5s)", error);
        } else if (error instanceof WebClientResponseException) {
            WebClientResponseException wcre = (WebClientResponseException) error;
            String body = wcre.getResponseBodyAsString();
            return new RuntimeException(
                String.format("API Giao Hàng Nhanh lỗi: %d - %s",
                    wcre.getRawStatusCode(), body), error);
        } else if (error instanceof WebClientRequestException) {
            return new RuntimeException("Lỗi kết nối mạng: " + error.getMessage(), error);
        }
        return error;
    }
}

// ===================== CONTROLLER =====================
@RestController
@RequestMapping("/api/orders")
@Slf4j
@RequiredArgsConstructor
public class OrderDeliveryController {

    private final DeliveryIntegrationService deliveryService;

    @PostMapping("/{orderId}/send-delivery")
    public Mono<ResponseEntity<DeliveryResponse>> sendToDelivery(
            @PathVariable String orderId,
            @RequestBody OrderDeliveryEvent event) {

        return deliveryService.sendOrderToDeliveryService(event)
            .map(response -> ResponseEntity.ok(response))
            .onErrorReturn(
                ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR)
                    .build());
    }
}

// ===================== CONFIGURATION =====================
@Configuration
public class WebClientConfig {

    @Bean
    public WebClient webClient() {
        // Client HttpClient async (Netty)
        HttpClient httpClient = HttpClient.create()
            .responseTimeout(Duration.ofSeconds(5))
            .compress(true)
            .wiretap(true); // Enable logging

        return WebClient.builder()
            .clientConnector(new ReactorClientHttpConnector(httpClient))
            .baseUrl("https://api.giao-hang-nhanh.vn")
            .build();
    }
}
```
